### 🎯 核心特性

- **Vue 3.4+ defineModel**: 21 个组件支持 defineModel API，提供更简洁的双向绑定
- **UTS 类型支持**: 完整的 TypeScript 类型定义，提供优秀的开发体验
- **主题定制**: 支持自定义主题色和组件样式
- **响应式设计**: 所有组件都支持响应式布局
- **无障碍支持**: 遵循 WCAG 2.1 标准

## 版本规范说明

### 版本号格式

采用 `主版本号.次版本号.修订号` 的格式：

- **主版本号**: 不兼容的 API 修改
- **次版本号**: 向下兼容的功能性新增
- **修订号**: 向下兼容的问题修正

### 变更类型

- **✨ 新增功能**: 新增的功能特性
- **🐛 Bug 修复**: 修复的问题和缺陷
- **⚠️ 破坏性变更**: 不向下兼容的变更（需要特别注意）
- **💄 样式优化**: UI 和样式的改进
- **⚡️ 性能优化**: 性能提升相关的改进
- **📝 文档更新**: 文档的新增或修改
- **🔧 配置调整**: 配置文件或构建流程的调整
- **♻️ 代码重构**: 不影响功能的代码重构
- **🎨 代码格式**: 代码格式化、注释等不影响功能的修改

### 破坏性变更标记

所有破坏性变更都会使用 ⚠️ 标记，并在变更说明中详细描述：

::: danger ⚠️ 破坏性变更
这是一个破坏性变更的示例说明。

**影响范围**: 描述受影响的组件或功能

**迁移指南**: 
1. 步骤一
2. 步骤二
:::
